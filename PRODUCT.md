# Product

## Register

product

## Users

Developers and AI-tool builders testing a small OpenRouter-backed chat surface locally. They want a focused place to send a prompt, see whether the configured model responds, and notice errors without digging through logs first.

## Product Purpose

This app proves a simple streaming chat path with Next.js, AI Elements-style components, the Vercel AI SDK, and OpenRouter. Success means the first screen is immediately usable, the configured model is used exactly as provided by environment, and the UI makes request state and failures easy to understand.

## Brand Personality

Calm, capable, direct. The interface should feel like a trustworthy product tool, not a marketing page or visual experiment.

## Anti-references

Avoid landing-page chrome, oversized hero treatment, decorative gradients, busy dashboard furniture, fake model metadata, and novelty controls that make the chat task feel less familiar.

## Design Principles

- Put the conversation first.
- Make system state visible without making it loud.
- Prefer familiar product patterns over novelty.
- Keep copy short and specific.
- Preserve the real backend contract over decorative UI claims.

## Accessibility & Inclusion

Target WCAG AA: readable contrast, visible focus states, keyboard-operable controls, reduced-motion support, and clear error messages that do not reveal secrets.
